02 — Logo

O logotipo, o Spark, e a regra zero.

O logotipo e o Spark nunca são gerados, desenhados ou reconstruídos a partir de descrição textual. É sempre o arquivo binário original que deve ser inserido na peça final.

Regra zero
Nunca usar um gerador de imagem (Canva, DALL·E, SVG escrito à mão, etc.) para produzir uma versão nova do logotipo ou do Spark — mesmo "só para preencher espaço". A regra é sobre proveniência do arquivo, não só sobre aparência final.

Mudança de direção

A VML está migrando de "Spark grande + logotipo pequeno em lockup" para logotipo como asset dominante, com o Spark em uso secundário e comedido. Se uma peça de referência antiga mostra Spark grande dominando a composição, ela está desatualizada frente a este guideline.

Hierarquia de uso

  1. Logotipo sozinho — uso encorajado em aplicações de propriedade/branded da VML. Forma preferencial.
  2. Logotipo + Spark separados — quando se deseja um elemento gráfico de marca adicional. Como elementos distintos, não lockup fundido.
  3. Spark sozinho — assinatura secundária (ex.: primeira/última página de um deck, rodapé de site).

Proporção logotipo × Spark (separados)

  • Alinhados no eixo X: Spark geralmente casa com 100% da altura do logotipo.
  • Alinhados no eixo Y: ponto de partida é Spark = 50% da altura do logo (logo grande).
  • Logo pequeno na composição: usar Spark a 75% do tamanho do logo.
  • Watch-out: Spark nunca maior que a altura do logotipo quando usados separados.

Arquitetura de marca

Marca-mãe → nomes com "VML" (endorsement) → nomes com linha de endosso → logos de aquisição com linha de endosso → programas/estúdios/práticas (sublinha "AT VML") → branding local de escritório (uso casual apenas, nunca substitui a assinatura primária).

Logos regionais com o símbolo Spark: usar exclusivamente Snowflake Sans Black.

Do
Usar logotipo preto ou branco na maioria das representações.
Don't
Preencher logotipo e Spark com cores diferentes na mesma peça. Criar logos ou lockups próprios — encaminhar ao time de Brand Design.
